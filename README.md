# Haatzmaut

מערכת ניהול מרפאה פנימית בשפה העברית (RTL) — SPA סטטי, ללא שרת.

## תכונות

| תכונה | פרטים |
|---|---|
| **לוח יומי** | תצוגה של יום בודד — שורות = חצאי שעה (08:00–20:00), עמודות = חדרים |
| **12 חדרים** | ניתן להוסיף, לערוך ולמחוק דרך מרחב האדמין |
| **חוצצי ימים** | לחצני יום (א׳–ה׳) עם תאריך ומספר הזמנות; החלפה מידית |
| **ניווט שבועי** | ← שבוע קודם / שבוע נוכחי / שבוע הבא → |
| **עיצוב מאוחדת** | צבעי מאוחדת – כחול וכתום (#0072BC ו-#F47B20) |
| **בחירת שעת סיום** | הזמנה נקבעת לפי שעת סיום ולא משך; צבע החדר מפוסק על כל משך ההזמנה |
| **לחיצה להזמנה** | לחיצה על תא ריק פותחת דיאלוג הזמנה (אדמין בלבד) |
| **גרירה ושחרור** | גרירת הזמנה לתא אחר עם בדיקת התנגשות (אדמין בלבד) |
| **מרחב אדמין** | ניהול חדרים, צוות, משתמשים; יצירת משתמש עם סיסמה ניבנית + איפוס על-פי בקשה |
| **שיוך משתמש לצוות** | משתמשי Staff משויכים לאיש צוות רשום, כולל ולידציה ומניעת שיוך כפול |
| **סנכרון לו"ז מתגלגל** | תבנית ברירת מחדל מסונכרנת אוטומטית לשבוע נוכחי + שני השבועות הבאים |
| **החלפת לו"ז לפי טווח** | בטעינת קובץ חדש ניתן לבחור החלפה לשבוע נוכחי / שבועות הבאים / שלושתם |
| **בקשות שינוי** | כל משתמש (כולל אדמין) יכול להגיש בקשה; הבקשה מסומנת ממתינה עד אישור/דחייה |
| **קידוד צבעים** | כרטיסיות הזמנה צבועות לפי צוות (ילדים / מבוגרים / נוער / זוגות / אדמין) |
| **רצועת צבע חדר** | כל הזמנה מקבלת פס צבע חדר לאורך כל משך ה-rowspan בטבלה |
| **התראות + Toast** | כל פעולה מוצגת כהתראה + הודעת toast |
| **שמירה מקומית** | כל המידע שמור ב-localStorage; שורד רענון דף |
| **ישיבות יום ראשון** | רשימת ישיבות קרובות/עבר + יבוא CSV/JSON (דובר, קהל יעד, פירוט, תאריך/שעה, קישור, קבצים) |
| **תקלות** | דיווח תקלה חופשי, גם כללי ולא רק לפי חדר |
| **תצוגת קליניקה חיה** | עמוד ייעודי להצגה במסך מרפאה: היום בלבד, שעה אחורה + 3 קדימה, סבב 10 חדרים כל 30 שניות + ספירה לאחור להחלפה הבאה |
| **תבניות CSV להורדה** | תבניות מוכנות ללו"ז, ישיבות, צוות וחדרים — זמינות מהממשק ומנתיב templates/ |
| **ייצוא נתונים נוכחיים** | הורדת לו"ז, צוות, ישיבות וחדרים כ-CSV עם הנתונים הנוכחיים ממש (כפתורי "הורדת X נוכחי") |
| **יבוא חדרים מ-CSV/JSON** | במרחב האדמין ניתן לייבא רשימת חדרים; בכפילות id יעדכן הרשומה הקיימת |
| **יבוא צוות מ-CSV/JSON** | במרחב האדמין ניתן לייבא רשומות צוות; בכפילויות נשמרת עדיפות לרשומת צוות שכבר משויכת למשתמש מערכת |
| **ניהול מסך תצוגה מאדמין** | האדמין יכול לשלוט על זמן החלפה בשניות, שעות אחורה/קדימה, מספר חדרים בכל סבב, והודעות זמניות או ללא הגבלה |

## הפעלה מקומית
```bash
cd /workspaces/Haatzmaut
python -m http.server 8000
```

ואז לפתוח בדפדפן: `http://localhost:8000`

עמוד תצוגת קליניקה למסך המתנה:
- `http://localhost:8000/display.html`

גישה לעמוד התצוגה זמינה:
- ממסך הכניסה
- מקישור עליון לאחר התחברות
- מתוך מרחב האדמין, דרך מודול "ניהול מסך תצוגה"

## משתמשי הדגמה
- ברירת מחדל: משתמשי מערכת נשמרים בתוך `localStorage`.
- נתיב התחברות legacy לפיתוח (`DEFAULT_CREDENTIALS`) זמין רק בהרצה מקומית עם `?devAuth=1`.
- דוגמה: `http://localhost:8000/?devAuth=1`

## אבטחה, בקרה וגיבוי

### מה נוסף בגרסה זו
- נעילת התחברות זמנית לאחר ניסיונות כושלים.
- פקיעת סשן אוטומטית לאחר חוסר פעילות.
- מגבלת גודל קובץ בייבוא + בדיקת סיומת קובץ.
- תצוגה מקדימה לפני ייבוא בפועל (לו"ז / חדרים / צוות / ישיבות).
- מסייע שלמות נתונים בתחילת טעינה (Integrity Assistant) עם תיקון אוטומטי מאושר משתמש.
- יומן בקרה (Audit Log) במרחב אדמין.
- ייצוא/שחזור גיבוי מלא כ-JSON.

### ניהול נתיב התחברות לפיתוח
- ב-Production אין להשתמש בנתיב legacy.
- לצורכי פיתוח בלבד:
	1. להריץ מקומית על `localhost` או `127.0.0.1`.
	2. להוסיף `?devAuth=1` ל-URL.

```

templates/   — קבצי תבנית CSV להורדה (לו"ז, ישיבות, צוות, חדרים)
```

## CSV/JSON Import

כל שורה / אובייקט חייב להכיל:
```
weekISO, day (0-4), roomId (או שם חדר), start (HH:MM), duration (דקות), staff, team
```

ניתן גם לטעון תבנית ללא `weekISO`; המערכת תשייך אותה אוטומטית לשבועים לפי טווח ההחלפה שנבחר.

### תבנית מוכנה להורדה
- `templates/bookings_template.csv`

### יבוא ישיבות יום ראשון

CSV/JSON ישיבות תומך בשדות:
```
speaker, audience(all|children|adults), specification/agenda, date(YYYY-MM-DD), time(HH:MM), link, files
```

לשדה `files` אפשר להזין רשימה מופרדת בנקודה-פסיק בקובץ CSV.

### תבנית מוכנה להורדה
- `templates/meetings_template.csv`

### יבוא צוות

CSV/JSON צוות תומך בשדות:
```
fullName/name, phone, email, role, team
```

בעת כפילויות, המערכת תשאיר בעדיפות את איש הצוות שכבר משויך למשתמש מערכת (`users.staffId`).

### תבנית מוכנה להורדה
- `templates/staff_template.csv`

### יבוא חדרים

CSV/JSON חדרים תומך בשדות:
```
id, name, tags
```

`id` — מזהה ייחודי (אופציונלי; אם לא מוזן, נוצר אוטומטית).  
`tags` — תגיות מופרדות בפסיק.  
בעת כפילות `id`, הרשומה הקיימת תוחלף.

### תבנית מוכנה להורדה
- `templates/rooms_template.csv`

### ייצוא נתונים נוכחיים

בנוסף לתבניות, ניתן להוריד את הנתונים הנוכחיים ממש:
| כפתור | תוכן הקובץ |
|---|---|
| הורדת לו"ז נוכחי (CSV) | כל הזמנות הלוח הנוכחי, כולל שם חדר |
| הורדת צוות נוכחי (CSV) | רשימת אנשי הצוות הרשומים |
| הורדת ישיבות נוכחיות (CSV) | כל ישיבות הצוות השמורות |
| הורדת חדרים נוכחיים (CSV) | רשימת החדרים הפעילים כולל תגיות |


## ניהול מסך תצוגה

במרחב האדמין ניתן להגדיר עבור [display.html](display.html):
- זמן החלפה בין קבוצות חדרים, בשניות
- כמה שעות אחורה וכמה שעות קדימה להציג
- כמה חדרים להציג בכל סבב
- הודעות למסך התצוגה למשך 5, 10, 15, 30, 60 דקות או ללא הגבלת זמן

כאשר יש כמה הודעות פעילות, חלון ההודעות במסך התצוגה גולל אותן אוטומטית.
